1. Set version in AssemblyInfo.cs
2. Build release in Visual Studio
3. Use VS Code to convert relevant markdown files (.md) under /docs into PDFs
	1. Markdown PDF Extension
	2. Open markdown file
	4. Right click, export to PDF
4. Run pack-release.bat
5. The pattern for {version} is: {Major}.{Minor}.{Patch}(-{PreRelease})?
	1. A version with a PreRelease is a beta release
6. The new release will be moved to the root folder (gobchatex-{version}.zip) along with a debug release (debugging symbols)
7. Upload a new release on github.com
	1. Release title is v{version}
	2. Description is used for patch notes
	3. Beta release? Check 'pre-release'
        4. Upload gobchatex-{version}.zip
	5. Updater will download the gobchatex-{version}.zip attachment

Note: releases are published on the fork github.com/Shuro/GobchatEx.
Note: CEF binaries are hosted as a separate one-time release (tag v2.0.0-cef)
      with assets Cef-107.1.12-{x64|x86}.7z. This release must exist for the
      first-run CEF download to work - see Module/Cef/Internal/CefInstaller.cs (CEF_URL).
